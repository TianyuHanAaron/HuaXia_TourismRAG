# HuaXia TourismRAG React Frontend

This is the public Xiaxia UI for HuaXia TourismRAG. It is a Vite-powered
React + TypeScript SPA backed by the existing FastAPI/Pydantic DTO layer.

## Local Development

Run FastAPI from the repository root, then start the frontend:

```bash
uv run uvicorn huaxia_tourismrag.main:app --reload
cd frontend
npm install
npm run dev -- --host 127.0.0.1
```

`npm run dev` syncs public UI assets from the canonical root `assets/`
directory before starting Vite.

For local API calls, create `frontend/.env.local`:

```env
VITE_API_BASE_URL=http://127.0.0.1:8000
```

## Scripts

- `npm run api:generate`: export FastAPI OpenAPI and regenerate Orval client code.
- `npm run assets:sync`: copy root assets into `frontend/public/assets/huaxia`.
- `npm run lint`: run ESLint.
- `npm run typecheck`: run TypeScript project checks.
- `npm run test`: run Vitest component/unit tests.
- `npm run test:e2e`: run Playwright app-shell checks.
- `npm run build`: sync assets, regenerate API types, typecheck, and build.

## Architecture Notes

The backend remains the source of truth for DTOs. Frontend API types are
generated by Orval and should not be edited by hand.

TanStack Query owns server state. Zustand is used only for UI state such as
language, active job/session ids, composer text, tab selection, and modal
visibility.

Do not put provider API keys in the browser bundle. Voice transcription,
generation, retrieval, and RAG citation validation all go through FastAPI.
